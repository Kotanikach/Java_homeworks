Практика #2
Что означает использование final при в данном объявлении?
try {
   ...
} catch (final Exception e) {
   ...
}

Использование ключевого слова final в объявлении переменной в блоке catch означает, что переменная не может быть изменена после ее инициализации. Это полезно, если вы хотите гарантировать, что ссылка на объект исключения не будет изменена в дальнейшем коде блока catch.

try {
    // Код, который может вызвать исключение
} catch (final Exception e) {
    // Здесь переменная e не может быть переназначена
    System.out.println("Произошло исключение: " + e.getMessage());
    // e = new Exception(); // Это вызовет ошибку компиляции
}

Таким образом, использование final в блоке catch обеспечивает защиту от изменения ссылки на объект исключения, что может быть полезно для обеспечения ясности кода и предотвращения ошибок.
